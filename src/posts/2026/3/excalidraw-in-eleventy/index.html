Excalidraw Diagrams in Eleventy

Published {% prettyDate date %}, {% readTime page %}

I love Excalidraw for quick diagrams. The hand-drawn aesthetic is charming and the tool is dead simple. But embedding those diagrams in a static site usually means shipping a hefty client-side bundle — or exporting PNGs by hand like some kind of animal.

What if we could just drop an .excalidraw file next to our post and have 11ty render it to inline SVG at build time? Turns out we can.

The shortcode

Here's the entire usage:

{%- raw %}{% excalidraw "./diagram.excalidraw" %}{% endraw %}

That's it. Point it at a .excalidraw file relative to your post, and the plugin reads the JSON, converts it to SVG, and inlines it directly into your HTML. No JavaScript ships to the browser.

How it looks

Here's an excalidraw diagram rendered at build time:

{% excalidraw "./diagram.excalidraw" %}

That SVG was generated during pnpm build. View source — there's no <script> tag, no web component, no client-side rendering. Just an inline SVG.

How it works

The plugin uses @moona3k/excalidraw-export which reimplements excalidraw's rendering using pure math from roughjs — no DOM, no canvas, no browser required. The whole conversion takes under 100ms per diagram.

The 11ty plugin itself is about 25 lines:

  1. Resolve the file path relative to the current post
  2. Read and parse the .excalidraw JSON
  3. Call renderToSvg() to get an SVG string
  4. Strip the fixed width/height and add responsive styles
  5. Return the inline SVG

Why not a web component?

I built a <wc-excalidraw> web component first. It works fine, but it ships @excalidraw/utils to every visitor (~1.4 MB) just to render a static diagram. Pre-rendering at build time means the diagram is already in the HTML — the browser just paints it.

For a blog, that tradeoff is obvious. Save the web components for things that actually need interactivity.
